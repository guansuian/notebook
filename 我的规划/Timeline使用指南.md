## 📝 第一步：两种核心语法

### 1. 极简模式：`timeline`

这种模式适合快速记录，使用 `+` 作为分隔符。每个事件**必须固定使用 3 个 `+`**。

- **格式要求：**
    
    - `+ [日期/时间]`
        
    - `+ [标题]`
        
    - `+ [具体内容描述]`
        

Markdown

````
```timeline
[line-3, body-2]
+ 2026-02-04
+ 🔴 未完成：Kafka 幂等性调试
+ 遇到了 Request Timeout 错误，需要排查 Broker 配置。链接：[[Kafka调优]]

+ 2026-02-03
+ ✅ 已完成：JWT 逻辑重构
+ 成功实现了 Refresh Token Rotation。
````

````

### 2. 明细模式：`timeline-labeled`
这种模式更具可读性，适合内容较多的任务描述。

* **格式要求：** 使用 `date:`, `title:`, `content:` 三个标签。

```markdown
```timeline-labeled
[line-4, body-3]
date: 2026-02-04
title: 🛠️ 开发进度：分片上传
content: 
1. 完成了前端 File.slice 切片。
2. [[Redis]] 状态位同步逻辑待测试。

date: 2026-02-02
title: 🏗️ 架构设计
content: 确定了使用 MinIO 作为文件存储后端。
````

```

## 🎨 第三步：样式自定义 `[ ]`
在代码块的第一行，你可以通过 `[ ]` 里的参数来改变外观。

| 参数类型 | 可选值 | 说明 |
| :--- | :--- | :--- |
| **线条 (line)** | `line-2` ~ `line-5` | 数字越大，中间的那条时间轴线越粗或样式不同。 |
| **主体 (body)** | `body-2` ~ `body-4` | 改变事件卡片的边框和阴影风格。 |
| **颜色 (color)** | `active-color-text-accent` | 使用你当前 Obsidian 主题的强调色。 |
| **错误色** | `active-color-text-error` | 适合标记“未完成”或“紧急”的时间轴。 |

## 💡 高级技巧：如何实现你的“任务回溯”

为了满足你**“上半部分未完成，下半部分已完成”**且能**链接笔记**的需求，建议采用以下排版规范：

1.  **使用换行：** 在 `content` 中使用 `<br>` 实现卡片内换行。
2.  **双向链接：** 直接在内容里写 `[[笔记名]]`，渲染后可直接点击。
3.  **状态标识：**
    * 日期后面加备注，例如 `date: 2026-02-04 (TODO)`。
    * 利用 Emoji 视觉区分：🔴 代表 Blocked，🟡 代表 进行中，🟢 代表 已完成。

> **避坑指南：**
> * 如果你在 `+` 后面直接换行而没有写新的 `+`，插件可能会解析失败。
> * 确保 `[` 和 `]` 是英文半角符号。
```
