### 1. 实战源码库（最高效的学习方式）

对于 Java 开发者，最快掌握 RBAC 的方法是**看成熟的开源项目怎么写**。不要自己造轮子，先看轮子。

- **首推：若依 (RuoYi) - 必看！**
    
    - **为什么推荐：** 这是国内 Java 圈公认的后台管理系统“教科书”。它的 RBAC 设计非常标准，涵盖了我们刚才讨论的所有面试题（多表设计、数据权限、注解控制）。
        
    - **去哪里找：** Gitee 或 GitHub 搜索 `RuoYi-Vue` (前后端分离版)。
        
    - **看什么：**
        
        - **SQL 文件：** 看 `sys_user`, `sys_role`, `sys_menu` 表结构。
            
        - **Mapper XML：** 看它是怎么做联表查询权限的。
            
        - **DataScopeAspect.java：** 搜这个类，看它是怎么用切面实现**数据权限**过滤的（这是高阶内容）。
            
- **备选：ElAdmin / Pig**
    
    - **ElAdmin：** 代码比若依更优雅一些，使用了 JPA，适合喜欢代码整洁的同学。
        
    - **Pig：** 如果你想看 **Spring Cloud 微服务** 架构下的 RBAC 怎么做（涉及 OAuth2），看这个。

### 4. 经典书籍（系统化理论）

虽然书更新慢，但如果你想通过面试，书里的**专业术语**很有用。

- **《Spring Security 实战》**：把这本书啃下来，RBAC 在你眼里就是透明的。

### 5. 你的学习路线图（建议）

按照这个顺序学，不要跳步：

1. **第一天：搞定数据库设计。**
    
    - 画出 User, Role, Permission, User_Role, Role_Permission 这 5 张表的 ER 图。
        
    - 弄清楚为什么需要关联表。
        
2. **第二天：搞定 SQL。**
    
    - 写一条 SQL：`给定一个 user_id，查出他所有的 permission_code`。这是一切的基础。
        
3. **第三天：结合 Spring Security。**
    
    - 学会怎么把查出来的 `permission_code` 塞给 Spring Security 的 `UserDetails`。
        
4. **第四天：注解控制。**
    
    - 学会用 `@PreAuthorize("hasAuthority('system:user:add')")` 保护接口。
        
5. **第五天（挑战）：数据权限。**
    
    - 去翻若依的源码，看懂它是怎么在 SQL 后面拼 `AND dept_id = 100` 的。
