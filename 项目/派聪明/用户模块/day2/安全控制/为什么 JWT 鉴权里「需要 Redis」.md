Redis 的作用不是“为了验证 JWT”，而是为了“弥补 JWT 天生不可撤销的缺陷”。
## 1️⃣ JWT 天生的问题是什么？

JWT 的核心特性是：**无状态（Stateless）**
什么意思？

```text
服务器不保存任何登录状态
是否登录 = 只看 token 本身
```

JWT 只靠三样东西判断合法性：
1. 签名是否正确
    
2. 是否过期
    
3. Claim 是否符合规则

👉 **它“不关心”这个 token 现在还应不应该被允许使用**

## 2️⃣ 现实系统真正需要什么？

真实业务里，后端一定需要这些能力：

- 手动踢人下线
    
- 封禁账号立即生效
    
- Token 被盗后立刻失效
    
- 修改权限后立刻生效
 

⚠️ **这些全部都是“状态控制”**

而 JWT **没有状态**

## 3️⃣ Redis 在这里扮演的角色

Redis 的本质作用只有一句话：

> **给 JWT 加一层“可控的状态门禁”**

JWT 负责：

- 身份不可伪造
    
- 内容不可篡改
    

Redis 负责：

- 这个 Token **现在** 还允不允许用
    

所以这是一个**分工**：

|组件|负责什么|
|---|---|
|JWT|身份真实性（你是谁）|
|Redis|使用合法性（你还能不能用）|
