[彩虹表](彩虹表.md)

加密的终极意义就是为了防止攻击者拿到明文密码。

对于加盐来说，主要是让攻击者构建彩虹表的时间成本和内存成本变得非常大。

## 为啥加盐能够让彩虹表失效呢？

彩虹表靠的是：**一次预计算，多次复用**。

- 无盐：攻击者可以预先算一张表，拿到任何人的哈希都能查。
    
- 有盐（每用户不同）：攻击者要为**每个 salt**单独准备一张表（等价于每个用户一张），存储和计算都爆炸。
    

所以彩虹表的优势（复用）没了，只剩下“针对单个用户的字典/暴力”，那还不如直接跑破解器。

## 关键补充：盐只能挡“预计算”，挡不住“离线猜测”

只要攻击者拿到了 **(salt, hash)**，就可以离线试：

```bash
for pwd in 常见密码字典:
    if H(pwd + salt) == hash:
        命中
```

所以真正的防线是两层叠加：

1. **随机 salt**：让预计算/彩虹表不复用
    
2. **慢哈希（bcrypt/Argon2）**：让“每次猜测”都很慢，从而暴力不划算
