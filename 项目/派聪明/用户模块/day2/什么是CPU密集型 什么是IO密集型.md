
# 什么叫「CPU 密集型」？

这是**操作系统 + 性能面试高频词**。

## 1️⃣ 定义一句话

> **CPU 密集型操作：主要消耗 CPU 计算能力，而不是等待 IO。**

---

## 2️⃣ 举几个典型 CPU 密集型操作

- JWT 签名验证（HMAC / RSA / ECDSA）
    
- 密码 hash（bcrypt / Argon2）
    
- JSON Web Token 的 crypto 运算
    
- 压缩 / 解压
    
- 图像处理
    
- 加解密
    

👉 特点：**算就完事了，没得等**

## 对比：IO 密集型（Redis 属于这类）

IO 密集型特点：

- 大部分时间在等：
    
    - 网络
        
    - 磁盘
        
    - 内存访问
        
- CPU 实际没怎么干活
    

Redis 查询：`CPU 算力消耗：极低 主要是内存 + 网络`
